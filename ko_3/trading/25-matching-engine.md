# 25. Matching Engine

### 오더북에서 주문이 매칭되는 4가지 방법

***

#### 🤝 직접 매칭 — 매수자와 매도자 만남

가장 단순한 케이스: YES 매수자와 YES 매도자가 있으면 → 양쪽이 직접 교환.

```
YES 매수자  →  USDC 지불  →  YES 토큰 수령
YES 매도자  →  YES 토큰 지불  →  USDC 수령
```

***

#### 🪙 새 토큰 민팅 — 두 사람이 각각 양쪽 매수

YES 매수자 **와** NO 매수자가 동시에 있고, 가격 합계가 **$1.00 이상**일 때 발생.

프로토콜이 양쪽의 USDC를 모아 새로운 YES + NO 쌍을 민팅해 각자 원하는 토큰을 나눠 줌.

**예시:**

* A가 YES를 $0.72에 매수
* B가 NO를 $0.30에 매수
* 합계: $1.02 → 프로토콜이 토큰 민팅, 잉여 $0.02는 프로토콜 수수료

> 💡 이 메커니즘으로 사전 유동성이 없어도 새 토큰이 생성됨.

***

#### 🔥 기존 토큰 소각 — 두 사람이 각각 양쪽 매도

민팅의 반대: YES 매도자 **와** NO 매도자가 동시에 있고, 가격 합계가 **$1.00 이하**일 때.

프로토콜이 YES + NO 쌍을 모아 소각하고 양쪽에 USDC를 돌려줌.

**예시:**

* A가 YES를 $0.50에 매도
* B가 NO를 $0.45에 매도
* 합계: $0.95 → 각자 설정 가격대로 USDC 수령, 부족한 $0.05는 청산 비용

> 💡 이 메커니즘으로 반대편 직접 매수자가 없어도 포지션을 청산할 수 있음.

***

#### 🎁 설정가보다 좋은 가격에 매칭

시스템은 반드시 설정가에 체결하지 않고, 가능한 한 **가장 유리한 가격**에 주문을 매칭하는 걸 우선함.

**예시:**

* YES 매수 주문을 $0.70에 냄 (최대 $0.70까지 지불 용의)
* 시장에 YES 매도자가 $0.50에 있음
* 시스템이 $0.50에 체결 → 즉시 **$0.20 환불**

```
지불 용의:       $0.70
실제 체결:       $0.50
환불 금액:       $0.20  ✅
```

> ⚡ 이는 limit order가 동의한 가격보다 비싸게 지불하는 일이 결코 없음을 의미 — 오직 더 싸질 수만 있음.
