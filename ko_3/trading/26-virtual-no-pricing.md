# 26. Virtual NO Pricing

### 왜 NO 토큰은 자체 유동성 풀이 없는가?

시스템에는 단 하나의 유동성 풀만 있음: **YES/USDC**. NO 토큰은 자체 풀이 없지만 — 시스템이 뒤에서 복잡한 부분을 처리하기 때문에 NO를 정상적으로 매매할 수 있습니다.

***

### NO 거래 시 뒤에서 벌어지는 일

#### **NO 매수** 시:

```
USDC 지불
  → 시스템이 YES + NO 쌍을 민팅
  → YES 부분을 유동성 풀에 매도
  → NO를 보유해 사용자에게 전달
NO 수령 ✅
```

#### **NO 매도** 시:

```
NO 지불
  → 시스템이 유동성 풀에서 YES 매수
  → YES + NO를 완전한 쌍으로 결합
  → 쌍을 USDC로 교환
USDC 수령 ✅
```

> 💡 위 과정은 모두 **단일 트랜잭션 내에서 즉시** 발생. 사용자 관점: USDC 넣고 NO 받음 — 또는 그 반대. 이 정도로 간단합니다.

***

### YES와 NO 가격 관계

두 토큰 가격의 합은 항상 **약 $1.00**:

```
NO 가격  ≈  $1.00 − YES 가격

예시:
  YES가 $0.70  →  NO는 약 $0.30
  YES가 $0.40  →  NO는 약 $0.60
```

**이 비율을 유지하는 것은?**

가격이 균형에서 벗어나면, 전문 트레이더(arbitrager)들이 즉시 개입해 차익을 챙깁니다 — 바로 그 행동이 가격을 올바른 비율로 되돌립니다.

**예시:** YES = $0.60, NO = $0.50이면 (합 $1.10 — 과열):

* 트레이더가 YES+NO 쌍을 $1.00에 매수
* 시장에 양쪽 매도해 $1.10 수령
* $0.10 이익 → 매도 압력이 가격을 $1.00로 되돌림

> ⚡ 이는 자동 균형 메커니즘입니다 — 관리자 불필요, 시장이 스스로 조정.
