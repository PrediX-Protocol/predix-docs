# 작동 방식

## 마켓 라이프사이클

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  생성    │ →  │  활성    │ →  │  만기    │ →  │  해결    │ →  │  수령    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                     │                                │
                   거래                          긴급 (7일)
                   가능                          또는 환불 모드
```

## 단계별 설명

### 1. 마켓 생성

관리자가 질문, 종료 시간, 오라클을 설정하여 마켓을 생성합니다:

```
"2026년 12월 31일까지 BTC가 $100,000를 초과할까?"
  → YES 토큰 + NO 토큰 + Uniswap v4 AMM 풀 ($0.50에 자동 초기화)
```

### 2. 거래

사용자는 거래를 통해 결과 토큰을 획득합니다:

- **분할**: 1 USDC 예치 → 1 YES + 1 NO 토큰 수령
- **Router 매수**: USDC 지불 → Smart Router가 CLOB + AMM에서 최적가 탐색 → YES 또는 NO 토큰 수령
- **지정가 주문**: CLOB에 특정 가격으로 주문 배치

### 3. 가격 발견

토큰 가격은 마켓의 확률 전망을 반영합니다:

| YES 가격 | 해석 |
| --------- | -------------- |
| $0.70 | 마켓이 YES 결과에 70% 확률로 예상 |
| $0.30 | 마켓이 YES 결과에 30% 확률로 예상 |

> ⚠️ **핵심 불변식**: YES 가격 + NO 가격 ≈ $1.00 (분할/병합 차익거래로 유지)

### 4. 해결

이벤트 발생 후:

1. 오라클 (Manual 또는 Chainlink)이 결과를 보고
2. 누구나 `resolveMarket(marketId)` 호출하여 최종 확정

### 5. 수령

승리 토큰 보유자는 토큰당 $1.00를 수령합니다:

```
YES 승리 시:
  1 YES 토큰 → $1.00 USDC
  1 NO 토큰  → $0.00

NO 승리 시:
  1 YES 토큰 → $0.00
  1 NO 토큰  → $1.00 USDC
```

## 실제 예시

Alice는 BTC가 $100K를 초과할 것이라 예상합니다. 현재 YES 가격: $0.65.

| 행동 | Alice 지불 | Alice 수령 | 손익 |
| ------ | ---------- | -------------- | --- |
| 100 YES @ $0.65 매수 | 65 USDC | 100 YES | — |
| BTC $100K 초과 ✅ | — | — | — |
| 100 YES 수령 | — | 100 USDC | **+35 USDC (54% 수익)** |

BTC가 $100K를 초과하지 않으면, Alice의 100 YES 토큰은 $0의 가치. 손실: 65 USDC.
