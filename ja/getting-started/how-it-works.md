# 仕組み

## マーケットライフサイクル

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  作成    │ →  │  アクティブ│ →  │  満期    │ →  │  解決    │ →  │  受取    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                     │                                │
                   取引                          緊急 (7日)
                   可能                          または返金モード
```

## ステップバイステップ

### 1. マーケット作成

管理者が質問、終了時間、オラクルを設定してマーケットを作成：

```
「2026年12月31日までにBTCは$100,000を超えるか？」
  → YESトークン + NOトークン + Uniswap v4 AMMプール（$0.50で自動初期化）
```

### 2. 取引

ユーザーは取引でアウトカムトークンを取得：

- **分割**: 1 USDCをデポジット → 1 YES + 1 NOトークンを受取
- **Router経由購入**: USDC支払い → Smart RouterがCLOB + AMMで最良価格を検索 → YESまたはNOトークンを受取
- **指値注文**: CLOBに特定の価格で注文を配置

### 3. 価格発見

トークン価格はマーケットの確率見通しを反映：

| YES価格 | 解釈 |
| --------- | -------------- |
| $0.70 | マーケットはYES結果に70%の確率と予想 |
| $0.30 | マーケットはYES結果に30%の確率と予想 |

> ⚠️ **重要な不変条件**: YES価格 + NO価格 ≈ $1.00（分割/統合の裁定取引で維持）

### 4. 解決

イベント発生後：

1. オラクル（ManualまたはChainlink）が結果を報告
2. 誰でも`resolveMarket(marketId)`を呼び出して確定

### 5. 受取

勝利トークン保有者はトークンあたり$1.00を受取：

```
YES勝利時:
  1 YESトークン → $1.00 USDC
  1 NOトークン  → $0.00

NO勝利時:
  1 YESトークン → $0.00
  1 NOトークン  → $1.00 USDC
```

## 実例

AliceはBTCが$100Kを超えると予想。現在のYES価格: $0.65。

| アクション | Alice支払い | Alice受取 | 損益 |
| ------ | ---------- | -------------- | --- |
| 100 YES @ $0.65購入 | 65 USDC | 100 YES | — |
| BTC $100K超過 ✅ | — | — | — |
| 100 YES受取 | — | 100 USDC | **+35 USDC (54%リターン)** |

BTCが$100Kを超えなかった場合、Aliceの100 YESトークンは$0の価値。損失: 65 USDC。
